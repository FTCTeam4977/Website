Gyro Library


  • What?
  • Why?
  • How?
  • Usage
  • Get it

  • The HiTechnic gyro is a sensor that will let you know how many degrees per second you are turning at the time you read the sensor. This library uses a method known as integration to convert the gyro degrees/second data to the absolute number of degrees you have turned.

  • Knowing how many degrees you have turned is very useful in autonomous. Many teams use timing to drive a distance or for turning during this period. Using timing to determine how long an action should be in autonomous has many disadvantages. Just because your robot took 2.3 seconds to turn when you timed it does not mean it will on the field. You can't account for differences in motors, battery level, or different resistance from the floor.

    With some extra math, you can use the heading information from the gyro to do exact turns and drive straight in autonomous.

  • This library uses a concept called integration to convert the gyro's degree/second value in to a heading in degrees. Integration is the act of adding an infinite series of values over time. To do this we need three things, the current sensor value, the time since we last checked, and a variable to store our results in.

    Our implementation of integration looks like this:
    gyroData.deg += dbc(gyroData.degsec, 1)*((nPgmTime-lastTime)/1000);

    Basically, it converts the difference in milliseconds since the last update to seconds and multiplies that by the current sensor value. The result of this is then added to the last result. In the end, you end up with the total amount of degrees the gyro has turned from the point it started at.

  • Before you include the gyro file, define GYRO as the sensor port your gyro is in. Ex: #define GYRO S2
  • Include the library file. It should be in the same folder as your code or in the includes folder. Ex: #include "FTC_Gyro.c"
  • Initalize the gyro. This should be called once. The robot must not be moved during the calibration. Ex: initGyro();
